## 演示准备

### 环境检查清单

- [ ] 前端服务运行正常 (http://localhost:3000)
- [ ] 后端 API 运行正常 (http://localhost:8000)
- [ ] 数据库已初始化 (544 位议员数据)
- [ ] 网络连接稳定
- [ ] 浏览器已打开并清除缓存

### 快速启动

```bash
# 一键启动所有服务
docker compose up --build -d

# 等待服务启动 (约 30 秒)
sleep 30

# 检查服务状态
docker compose ps

# 初始化数据 (首次运行)
docker compose exec backend python3 -m app.etl.import_real_members
docker compose exec backend python3 -m app.etl.import_fec_data
docker compose exec backend python3 -m app.etl.import_holdings
docker compose exec backend python3 -m app.etl.import_congress_profiles
```

## 演示流程

### 1. 开场介绍 (2 分钟)

**开场白:**

> "大家好，今天我向大家展示一个美国国会利益关联图谱系统。这个系统通过可视化方式，帮助研究者分析美国参众两院议员的背景、利益关系和政治立场。"

**核心价值:**

- 数据驱动的政治研究工具
- 可视化复杂利益关系
- 支持多维度分析

### 2. 系统概览 (3 分钟)

**访问首页:**
- 打开 http://localhost:3000
- 展示议员列表和搜索功能
- 说明按议院和党派分类

**关键数据:**
- 544 位现任议员
- 49 个委员会
- 20,937 个竞选委员会
- 3,637 位捐赠者

### 3. 核心功能演示 (10 分钟)

#### 3.1 议员搜索和筛选

**演示步骤:**
1. 在搜索框输入 "Biden" 或 "Trump"
2. 使用筛选器选择 "参议院" + "民主党"
3. 展示实时搜索结果

**话术:**
> "系统支持按姓名、议院、党派、届次等多维度筛选，帮助快速定位目标议员。"

#### 3.2 议员详情页

**演示步骤:**
1. 点击任意议员进入详情页
2. 展示个人信息卡片
3. 查看委员会任职
4. 展示对华立场分析

**话术:**
> "每位议员都有详细的个人资料，包括政治立场、委员会任职、对华态度等关键信息。"

#### 3.3 利益关系图谱

**演示步骤:**
1. 在议员详情页点击 "图谱" 标签
2. 展示议员的利益关系网络
3. 演示节点展开功能
4. 展示不同关系类型的可视化

**话术:**
> "图谱功能将复杂的政治利益关系可视化，支持交互式探索，帮助发现隐藏的关联。"

#### 3.4 竞选献金分析

**演示步骤:**
1. 点击 "献金" 标签
2. 展示竞选委员会信息
3. 查看捐赠者列表
4. 分析资金流向

**话术:**
> "系统集成了 FEC 竞选献金数据，可以追踪政治资金的来源和流向。"

#### 3.5 持股披露

**演示步骤:**
1. 点击 "持股" 标签
2. 展示议员的资产披露
3. 查看持股详情和价值区间

**话术:**
> "持股披露功能展示了议员的财务利益，有助于识别潜在的利益冲突。"

#### 3.6 对华立场分析

**演示步骤:**
1. 点击 "档案" 标签
2. 展示对华立场摘要
3. 查看核心政治主张
4. 展示综合评价

**话术:**
> "系统对每位议员的对华立场进行了分析，帮助研究者了解国会中的对华态度分布。"

### 4. 高级功能演示 (5 分钟)

#### 4.1 图谱控制参数

**演示步骤:**
1. 展示图谱控制面板
2. 切换不同参数:
   - `include_profile_facts`: 履历事实
   - `include_finance`: 献金节点
   - `include_holdings`: 持股节点
3. 展示图谱变化

**话术:**
> "图谱支持多种控制参数，可以根据研究需求定制显示内容。"

#### 4.2 数据导出

**演示步骤:**
1. 展示 API 文档 (http://localhost:8000/docs)
2. 演示 Markdown 简报生成
3. 展示数据格式

**话术:**
> "系统提供完整的 API 接口，支持数据导出和二次开发。"

### 5. 技术架构介绍 (3 分钟)

**架构图:**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Frontend  │────▶│   Backend   │────▶│  PostgreSQL │
│  React 18   │     │   FastAPI   │     │    15      │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │    Neo4j    │
                    │   Graph DB  │
                    └─────────────┘
```

**技术栈:**
- 前端: React 18, TypeScript, Ant Design, AntV G6
- 后端: Python 3.10+, FastAPI, SQLAlchemy
- 数据库: PostgreSQL 15, Neo4j 5
- 部署: Docker Compose

### 6. 总结和问答 (2 分钟)

**总结要点:**

1. **数据全面**: 覆盖 544 位现任议员的完整信息
2. **可视化强**: 交互式图谱展示复杂关系
3. **分析深入**: 多维度分析政治利益
4. **易于部署**: Docker 一键启动
5. **开源开放**: 完整代码和文档

**话术:**
> "这个系统为政治研究提供了强大的数据工具，帮助研究者更高效地分析国会政治。感谢大家的聆听，现在欢迎提问。"

## 常见问题回答

### Q1: 数据来源是什么？

**A:** 数据来自多个公开来源:
- unitedstates/congress-legislators (CC0-1.0)
- FEC.gov 竞选献金数据 (Public Domain)
- 国会财务公开报告 (Public Domain)
- Wikipedia 履历信息 (CC BY-SA 4.0)

### Q2: 数据更新频率？

**A:** 目前是静态数据快照，计划未来支持:
- 定期数据同步
- 实时数据更新
- 历史数据版本

### Q3: 如何保证数据准确性？

**A:** 采取多重措施:
- 使用权威数据源
- 交叉验证多个来源
- 标注数据来源和时间
- 提供纠错反馈机制

### Q4: 支持哪些分析场景？

**A:** 主要应用场景:
- 政治学研究
- 政策分析
- 游说关系追踪
- 利益冲突识别
- 对华政策研究

### Q5: 技术难度高吗？

**A:** 部署简单，使用友好:
- Docker 一键部署
- 图形化界面
- 完整 API 文档
- 详细部署指南

### Q6: 数据安全如何保障？

**A:** 仅使用公开数据:
- 所有数据来自公开来源
- 无敏感个人信息
- 符合数据使用许可
- 开源代码可审计
